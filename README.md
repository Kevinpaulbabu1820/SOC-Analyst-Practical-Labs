# Example Lab Investigation Workflow

Each lab follows a structured investigation process similar to **SOC analyst procedures**.

### Objective

Investigate the triggered security alert and determine whether it represents a **true positive security incident or a false positive**.

### Investigation Steps

1. Review the alert generated by the SIEM or monitoring system
2. Analyze related system or authentication logs
3. Identify suspicious patterns or indicators of compromise (IOCs)
4. Validate the activity using threat intelligence or log correlation
5. Determine whether the alert represents malicious activity

---

# Detection Logic Example

Example detection rule for identifying a **brute force login attack**:

* Multiple failed authentication attempts
* Repeated login attempts from the same IP address
* Multiple accounts targeted within a short time frame

These patterns may indicate **credential brute forcing or password spraying attacks**.
