# 🚨 SOC342 – CVE-2025-53770 SharePoint ToolShell Auth Bypass & RCE

## 📌 Overview

This project documents the investigation of a **critical security alert (SOC342)** involving exploitation of **CVE-2025-53770** in **Microsoft SharePoint Server**.

The vulnerability allows attackers to:

* Bypass authentication
* Execute remote code
* Extract sensitive cryptographic keys

This case demonstrates a **multi-stage real-world attack**, including initial exploitation, persistence, and credential/key extraction.

---

## ⚠️ Vulnerability Summary

![Case Image](/LetsDefend/SOC342%20CVE/Case%20Details.png)

* **Type:** Authentication Bypass + Remote Code Execution
* **Impact:** Unauthorized access, command execution, data exposure
* **Severity:** Critical (actively exploited)

**In simple terms:**
An attacker can trick SharePoint into running malicious code **without logging in**.

---

## 🔍 Investigation Workflow

### 1️⃣ Log Analysis

* Identified suspicious **unauthenticated POST request**
* Target endpoint:

  ```
  /_layouts/15/ToolPane.aspx
  ```
* Key indicators:

  * Spoofed `Referer`: `/layouts/SignOut.aspx`
  * Large encoded payload (~7699 bytes)
  * No authentication headers

➡️ Matches known exploitation pattern of CVE-2025-53770

---

### 2️⃣ Threat Intelligence Check

![Checking Virus](/LetsDefend/SOC342%20CVE/VirusTotal.png)

* Tool used: VirusTotal
* Malicious IP: `107.191.58.76`
* Detection score: **11/94**

➡️ Indicates suspicious infrastructure used in attack

---

### 3️⃣ EDR Telemetry Analysis

Detected execution of:

```powershell
PowerShell - decode and deploy malicious ASPX
```

* Payload revealed a **webshell**
* Used for command execution and persistence

---

## 🧠 Webshell Analysis

### Key Behavior:

* Uses **.NET reflection**
* Extracts sensitive configuration:

```csharp
MachineKeySection → ValidationKey | DecryptionKey
```

### Why this matters:

* Enables **forging authentication cookies**
* Allows **session hijacking**
* Helps attacker maintain long-term access

---

## ⚙️ Post-Exploitation Activity

### 🔧 Command Execution

```cmd
csc.exe /out:C:\Windows\Temp\payload.exe payload.cs
```

➡️ Compiles malicious C# code on the system

---

### 📂 Webshell Deployment

```cmd
spinstall0.aspx
```

Location:

```
SharePoint LAYOUTS directory
```

* Acts as a **malicious downloader**
* Fetches payloads from attacker-controlled server

---

### 🔐 Final Stage – Key Extraction

```powershell
[System.Web.Configuration.MachineKeySection]::GetApplicationConfig()
```
![Command](/LetsDefend/SOC342%20CVE/Command.png)

➡️ Extracted:

* ValidationKey
* DecryptionKey
* Encryption settings

**Impact:**

* Forge authentication tokens
* Decrypt sensitive data
* Escalate privileges

## 🧾 Final Analysis

✔️ **True Positive Alert**

### Attack Chain:

1. Exploit SharePoint vulnerability
2. Bypass authentication
3. Deploy webshell
4. Execute PowerShell payload
5. Extract cryptographic keys

**In simple terms:**
The attacker gained access → planted a backdoor → stole security keys → prepared for long-term control.

---

## 🛡️ Mitigation & Recommendations

* Apply Microsoft security patches immediately
* Restrict access to SharePoint endpoints
* Monitor:

  * PowerShell activity
  * ASPX file creation
* Block malicious IPs
* Enable EDR alerting for:

  * `csc.exe` execution
  * Suspicious webshell behavior
