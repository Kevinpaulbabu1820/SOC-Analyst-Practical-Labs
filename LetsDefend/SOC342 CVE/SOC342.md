# Project Title: LetsDefend SOC Lab Walkthrough  
## Lab: SOC342 CVE-2025-53770 SharePoint ToolShell Auth Bypass & RCE  
## Platform : LetsDefend  

### Incident Title : SharePoint ToolShell Authentication Bypass & RCE  
### Incident ID : SOC342  
### Date : Jul 22 2025  

### Incident Description :  
- A critical alert was triggered involving exploitation attempts against Microsoft SharePoint Server using CVE-2025-53770.
- The attacker attempted to bypass authentication, deploy a malicious webshell, and extract cryptographic machine keys from the server.

---

## Alert Details  
#### Level : SOC Analyst  
#### Source IP Address : 107.191.58.76  
#### Target Application : Microsoft SharePoint Server  

---

## Investigation Steps  

### Analysis of Network Logs  
#### Key Observation  
- Suspicious unauthenticated POST request identified.
- Targeted endpoint:

```text
/_layouts/15/ToolPane.aspx
```

- Request contained:
  - Spoofed Referer header: `/layouts/SignOut.aspx`
  - Large encoded payload (~7699 bytes)
  - No authentication headers

#### Observation:
- Activity matches publicly known exploitation patterns associated with CVE-2025-53770.

---

### Threat Intelligence Investigation  

#### Tools Used  
- VirusTotal  

#### Observation:
- Malicious IP address `107.191.58.76` flagged as suspicious.
- Detection score identified malicious infrastructure activity.

---

### EDR Telemetry Analysis  

#### Observation:
- PowerShell activity detected attempting to decode and deploy a malicious ASPX payload.
- Payload behavior revealed deployment of a webshell for persistence and command execution.

```powershell
PowerShell - decode and deploy malicious ASPX
```

---

### Webshell Analysis  

#### Key Observation  
- Webshell leveraged .NET reflection techniques.
- Attempted extraction of sensitive SharePoint machine keys.

```csharp
MachineKeySection → ValidationKey | DecryptionKey
```

#### Observation:
- Extracted keys could allow:
  - Forging authentication cookies
  - Session hijacking
  - Persistent unauthorized access

---

### Post-Exploitation Activity  

#### Command Execution  

```cmd
csc.exe /out:C:\Windows\Temp\payload.exe payload.cs
```

#### Observation:
- Malicious C# payload compiled directly on the victim system.

---

### Webshell Deployment  

```cmd
spinstall0.aspx
```

#### Observation:
- Webshell deployed inside SharePoint LAYOUTS directory.
- Used as a malicious downloader for additional payload retrieval.

---

### Cryptographic Key Extraction  

```powershell
[System.Web.Configuration.MachineKeySection]::GetApplicationConfig()
```

#### Observation:
- Attacker extracted:
  - ValidationKey
  - DecryptionKey
  - Encryption settings

#### Impact:
- Authentication token forgery
- Sensitive data decryption
- Privilege escalation

## Remediation  
- Apply Microsoft SharePoint security patches immediately.
- Restrict access to exposed SharePoint endpoints.
- Monitor:
  - PowerShell activity
  - ASPX file creation
  - Suspicious csc.exe execution
- Block malicious IP address `107.191.58.76`
- Enable EDR alerting for:
  - Webshell activity
  - Unauthorized PowerShell execution
  - Machine key access attempts

---

## Conclusion  
- The attacker successfully attempted a multi-stage exploitation chain involving authentication bypass, remote code execution, webshell deployment, and cryptographic key extraction.
- Immediate remediation and continuous monitoring are essential to prevent long-term persistence and compromise within SharePoint environments.
